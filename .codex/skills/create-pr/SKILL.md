# プルリク実装

- `#` で始まる番号が指定されたら、GitHub の Issue 番号なので、それを読んで実装してください。
- README.md は書き換えないでください。

# ブランチ名

- プルリクのブランチは Issue が指定されていたら `issue_番号` にしてください。先頭に `codex/` を付ける必要は無いです。
- そうでないならば、任意にブランチ名を付けてください。`fix/` や `feature/` のようなプレフィックスは不要です。

# ブランチ作成方法

- develop ブランチに切り替え、リモートから最新のものを pull してから、プルリクのブランチを作成してください。

# プルリクタイトル

- Issue が指定されていたら、それと同じにしてください。
- Issue が指定されていない場合は、内容を表す日本語タイトルを付けてください。

# プルリク本文

- プルリク本文は .github/PULL_REQUEST_TEMPLATE.md に沿ってください。
